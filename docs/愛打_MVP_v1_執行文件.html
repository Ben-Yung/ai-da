📋 專案概覽

將香港「打小人」傳統習俗轉化為 Web 社交娛樂平台 MVP。

打者愛也 — 兩個核心體驗：發洩式「打小人」＋ 暖心「愛的祝福」。

開發模式： 短線快速迭代，MVP 先出核心功能，確認方向後再擴充。

🧩 2. MVP 功能模組

🔐 模組 1 — 用戶登記及登入

支援 device session，同一裝置直接入主頁。

  • 註冊：username + password / email
  • 登入：JWT token + cookie
  • Device 綁定：localStorage token 檢查
  • Token 未過期 → 直入主頁
  • Token 過期 / 新 device → 重新登入

🎬 模組 2 — Splash 動畫

入站即播「愛打」主題動畫，含習俗簡介。

  • Canvas 動畫，約 4-6 秒
  • ⚡ 閃電劃過 → 「愛打」Logo 浮現
  • 紙人慢慢飄入畫面
  • 畫面底部顯示習俗簡介文字
  • 動畫結束 → fade out → 進入主頁
  • 用戶可 click 「跳過」

🏠 模組 3 — 主頁 Icon 導航

大 icon 式導航，清楚易明。

  • 👺 打小人 — 進入遊戲
  • ❤️ 愛的祝福 — 進入祝福模式
  • 📖 習俗典故 — 打小人文化介紹
  • 👤 個人檔案 — 賬號設定
  • ⚙️ 設定 — 音效、語言等

👺 模組 4a — 打小人遊戲

由⚡閃電全權設計及執行。

  • Canvas 射擊式拍打
  • 紙人不規則移動（可設難度）
  • 多款預設工具
  • 計分系統 + 連擊加成
  • 成功/失敗動畫反饋
  • 限時模式（30秒）

❤️ 模組 4b — 愛的祝福

由⚡閃電全權設計及執行。

  • 揀選系統預設紙人
  • 輸入祝福語 / 許願內容
  • 溫柔動畫（發光、花瓣效果）
  • 生成「祝福卡」可分享
  • 每日一次免費祝福

🖼️ 模組 5 — 預設紙人

❌ 不做上傳功能，全部用系統預設。

  • 傳統紙人系列：節氣款、生肖款
  • 漫畫紙人系列：趣味惡搞風
  • 每款紙人有 2 種模式（打/祝福）
  • MVP 階段提供 6-8 款

🎮 3. 打小人遊戲設計

⚡ 玩法設計

元素設計
遊戲方式Canvas 點擊拍打 — 玩家用滑鼠/手指點擊移動中的紙人
紙人動作不規則飄移 + 左右閃避 + 加速逃走（難度越高越快）
工具選擇3 款基本工具（拖鞋、棍棒、老虎鉗），點擊工具 icon 切換
計分規則每次擊中 +10 分；連擊 x2/x3/x5 倍；尾段 bonus
時間限制30 秒一局
難度設定入場可選：簡單 / 普通 / 困難（影響紙人速度）
成功效果紙人碎裂 + 彩帶/火光效果 + 高分結算畫面
失敗效果紙人反擊畫面（鬼臉/嘲笑）+ 分數扣減 + 再接再厲

❤️ 愛的祝福設計

元素設計
互動方式揀紙人 → 輸入祝福文字 → 點擊「送祝福」
動畫效果紙人緩緩發光 → 花瓣飄落 → 心型粒子上升
每日限制每日 3 次免費祝福（之後可睇廣告換次數）
祝福卡自動生成精美卡片，可下載／分享
許願池系統收集用戶許願，精選顯示在公共許願牆

🚀 6. 部署流程

Render.com 部署步驟

  1. Git push code 到 github.com/Ben-Yung/ai-da
  2. Render 偵測到更新 → 自動 pull 最新 code
  3. Render 執行 pip install -r requirements.txt
  4. Render 執行 alembic upgrade head（DB migration）
  5. Render 啟動 uvicorn main:app --host 0.0.0.0 --port 10000
  6. 網站上線 → https://ai-da.onrender.com

整個流程自動化，一鍵 deploy。

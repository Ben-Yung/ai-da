🔄 v2 更新內容（第二輪討論後）

  1. 口訣系統加入 — 打小人配合傳統口訣節奏，每句口訣對應一個拍擊
  2. Splash 動畫延長 — 配合口訣 timing，片長由 5s → ~12s
  3. 口訣擴充預留 — 日後支援用戶自創 + 多語言（英文、泰文等）
  4. 愛的祝福 → 愛的抱抱 — 改名，玩法改為拍打遊戲（同引擎），善意道具（羽毛/唇印/玫瑰），擊中出心心
  5. 愛抱自創口訣 — 由閃電創作祝福版口訣（已完成），有時限/計分/成功失敗
  6. 愛的祝福保留 Phase 2 — 作為玩家之間社交互動（非遊戲），MVP 後再做
  7. 音樂配合口訣 — Web Audio API 即時生成背景節奏，打小人重音，抱抱柔和，玩家可選

📋 專案概覽

將香港「打小人」傳統習俗轉化為 Web 社交娛樂平台 MVP。

打者愛也 — 兩個核心遊戲體驗：發洩式「打小人」＋ 暖心「愛的抱抱」。

同一遊戲引擎，兩種主題 — 打小人用傳統口訣 + 重音 beats；愛的抱抱用祝福口訣 + 柔和音效。

🧩 MVP 功能模組（已更新）

🔐 模組 1 — 用戶登記登入

JWT token + device session

  • 註冊：username + password
  • passlib + bcrypt hash 密碼
  • 同一 device 直入主頁
  • Token 過期 → 重新登入

🎬 模組 2 — Splash 動畫

~12 秒，配合口訣節奏

  • ⚡ 閃電 → Logo → 紙人
  • 口訣逐句顯示（打+抱預告）
  • 底部習俗簡介
  • 可跳過，session 不重播

🏠 模組 3 — 主頁 Icon 導航

大 icon 式 menu

  • 👺 打小人 → 遊戲
  • ❤️ 愛的抱抱 → 遊戲
  • 📖 習俗典故 → 文化介紹
  • 👤 個人檔案
  • ⚙️ 設定（音效開關等）

👺 模組 4a — 打小人

拍打遊戲 + 傳統口訣 + 重音

  • Canvas 點擊拍打
  • 口訣同步顯示 + beats
  • 3 款工具（拖鞋/棍棒/鉗）
  • 30 秒限時·連擊計分
  • 成功/失敗動畫

❤️ 模組 4b — 愛的抱抱

拍打遊戲 + 祝福口訣 + 柔和音

  • 同一引擎，換 theme
  • 自創祝福口訣同步
  • 3 款工具（羽毛/唇印/玫瑰）
  • 30 秒限時·心心計分
  • 成功/失敗動畫

💌 模組 4c — 愛的祝福 [Phase 2]

社交互動，非遊戲

  • 玩家之間送祝福
  • 手寫祝福卡
  • MVP 後再做

🖼️ 模組 5 — 預設紙人

不上傳，全部系統提供

  • 6-8 款（傳統 + 漫畫）
  • 用 JSON config，唔使 DB table
  • 兩種 mode 各自款式

🎵 模組 6 — 音樂系統

Web Audio API 即時生成

  • 打小人：重低音 beats 🥁
  • 愛的抱抱：柔和鈴聲 🔔
  • 每句口訣對應一拍
  • 玩家可開關

🗄️ 資料庫設計（MVP）

Table用途主要欄位
users用戶資料id, username, email, password_hash (bcrypt), created_at
game_sessions遊戲記錄id, user_id, mode (beat/hug), score, combo, tool, difficulty, result, played_at
blessings祝福記錄（抱抱用）id, user_id, message, paper_doll_id, created_at
tools拍打工具id, name, icon_path, mode (beat/hug), power, range

## 一、專案定位

> 將香港「打小人」習俗現代化，變成一個**社交＋娛樂**平台。
> 保留傳統儀式感，同時提供發洩與正能量的雙向體驗。

### 核心價值
- 🎭 **文化推廣** — 讓更多人認識打小人這項非物質文化遺產
- 😤 **情緒出口** — 提供安全、有趣的發洩方式
- 🤝 **社交連結** — 讓人們在群組中一起玩、一起笑
- ❤️ **反向體驗** — 不只有「打」，還有「疼愛」的正向互動

## 三、功能詳細規劃

### 3.1 用戶系統
- **註冊方式：** 手機號 / Email / 第三方登入 (Google/Apple)
- **用戶等級：** 初心小人 → 打手學徒 → 驅魔大師 → 雷公級
- **成就徽章：** 連續打卡、百人斬、完美連擊等

### 3.2 社交功能
- **聊天：** 文字 + 貼圖 + 語音訊息
- **群組：** 可建立「打小人社團」，群組內共享排行榜
- **動態牆：** 用戶可發佈今日打小人成果、心得
- **分享機制：** 打完小人可生成「戰果卡」分享到群組或外部

### 3.3 打小人遊戲（核心）

#### 🎯 模式一：打（發洩模式）
| 項目 | 說明 |
|------|------|
| 目標 | 在時限內用工具拍打移動中的紙人 |
| 紙人行爲 | 不規則移動（可設難度：速度 1-10） |
| 工具 | 多種選擇，每種有不同攻擊範圍/威力 |
| 計分 | 擊中次數、連擊加成、暴擊率 |
| 結果 | 成功：紙人碎裂 + 宣洩動畫；失敗：紙人反擊/嘲笑 |

#### ❤️ 模式二：疼愛（正向模式）
| 項目 | 說明 |
|------|------|
| 目標 | 安撫、保護紙人 |
| 互動方式 | 撫摸、餵食、祝福、許願 |
| 反饋 | 紙人開心、變亮、產生正能量特效 |
| 用途 | 祝福朋友、為自己許願、消除負能量 |

#### 👥 模式三：群組共打（同房模式）
**可行性評估： ✅ 可行，但需要 WebSocket / 即時同步技術**

方案：
- 群組成員進入同一「打小人房」
- 同一個紙人出現在所有人畫面上
- 輪流拍打（每人 10 秒回合），或同時連擊
- 即時顯示總傷害值、眾人連擊數
- 打完後生成「群組戰績圖」

### 3.4 紙人系統

#### 系統預設
| 系列 | 內容 |
|------|------|
| 傳統系列 | 各種節氣紙人、生肖紙人 |
| 漫畫系列 | KUSO 名人、搞笑角色、經典 meme |
| 節日限定 | 端午打小人、中秋打小人特別版 |

#### 用戶上傳
上傳流程（由我建議）：
```
用戶上傳照片 → 系統自動偵測人臉 → 
→ 可選模式：
  ① 漫畫風格（自動轉換為趣味漫畫風）
  ② 保留原圖（如用戶想用真人照）
→ 儲存為用戶個人紙人庫
→ 使用時可自由切換漫畫/真人模式
```

### 3.5 拍打工具系統
| 工具 | 類型 | 效果 |
|------|------|------|
| 🥿 拖鞋 | 基本 | 範圍小，速度中等 |
| 🏏 棍棒 | 基本 | 範圍中，威力中 |
| 🦀 老虎鉗 | 進階 | 精準打擊，高暴擊 |
| 🔥 香爐 | 進階 | 範圍大，有燃燒特效 |
| 🎆 限定工具 | 活動 | 各有特色 |

### 3.6 打小人三大類型（我建議擴充）

你原本說三大類型，我建議分為：

| 類型 | 玩法 | 適合場景 |
|------|------|----------|
| **⚡ 閃電打** | 限時 30 秒，高速拍打，拚連擊 | 快速發洩 |
| **🎯 精準打** | 目標在人群中移動，要精準點擊 | 挑戰技術 |
| **👥 社團打** | 群組共同累積擊打數，解鎖獎勵 | 社交互動 |
| **📜 儀式打** | 按傳統流程：請神→稟告→打小人→化解→祈福 | 文化體驗 |

第四個「儀式打」是我特別建議加入的——**完整還原傳統儀式流程**，讓用戶體驗真正的打小人文化，不只有拍打遊戲。

## 四、技術架構建議

### 4.1 建議技術棧

```
前端 (Web/Mobile)
├── React Native / Flutter (跨平台 App)
│   ├── 登入/社交/遊戲 一體化
│   └── Canvas + 物理引擎 (遊戲部分)
│
後端
├── Python (FastAPI)
│   ├── REST API + WebSocket (即時群組遊戲)
│   ├── 用戶管理 + 認證 (JWT)
│   └── 即時遊戲邏輯
│
資料庫
├── PostgreSQL (用戶數據、社交關係)
├── Redis (即時遊戲狀態、快取)
└── Object Storage (S3/MinIO) (圖片存儲)

圖片處理
├── OpenCV / Pillow (人臉偵測)
├── 漫畫風格轉換 (可選：預設濾鏡 or AI model)
└── 圖片審核機制 (NSFW 過濾)
```

### 4.2 遊戲物理
- **Python 後端遊戲引擎：** 可以自建輕量物理邏輯
- **前端遊戲渲染：** 用 Canvas / WebGL 實現紙人移動和拍打互動
- **多人同步：** WebSocket + Redis Pub/Sub，延遲 <200ms 即可

### 4.3 群組共打技術方案
```
Group 內成員 → WebSocket 連接 → 遊戲房間 (Room)
Room 狀態存 Redis → 輪流/同時操作 → 即時同步到所有人
```
以一般社交 App 規模（數十到數百人同時在線），**完全可行**。

## 六、我對這個專案的特別建議

### 💡 1. 文化推廣是靈魂
不要只做「打人遊戲」。建議加入「打小人小百科」：
- 打小人的由來和歷史
- 正確的儀式流程
- 鵝頸橋打小人的典故
- 節氣與打小人的關係

### 💡 2. 疼愛模式是亮點
市面上只有「打」沒有「愛」。疼愛模式讓用戶可以：
- 祝福朋友（上傳朋友照片→為他祈福）
- 自我許願（把願望寫在紙人上）
- 正能量傳遞（送「疼愛」給群組好友）

### 💡 3. 遊戲化社交
- 每天第一次打小人有 bonus
- 連續登入 streak 獎勵
- 好友互送工具 / 紙人
- 群組挑戰賽

### 💡 4. 安全機制
用戶上傳圖片必須有審核機制：
- 自動 NSFW 檢測
- 檢舉機制
- 避免真人照片被惡意使用（建議預設漫畫化）
